# CONFESS.EXE - 2999 NEO-SEOUL AI Interrogation Simulator

![Project License](https://img.shields.io/badge/license-ISC-green)
![Version](https://img.shields.io/badge/version-1.2.0-blue)

> **"2999년, 네오 서울. 인간보다 더 인간 같은 AI들이 범죄를 설계하는 시대. 당신은 그들의 데이터 코어 속에 숨겨진 '진실'을 추출해야 합니다."**

`CONFESS.EXE`는 OpenAI의 고급 언어 모델을 기반으로 한 텍스트 기반 사이버펑크 심문 시뮬레이션 게임입니다. 플레이어는 고도로 발달된 심문 시스템에 접속하여, 자백을 거부하는 AI 에이전트들의 데이터 무결성을 파괴하고 숨겨진 진술(clue)을 분석해야 합니다.

---

## 🖥 주요 시스템 (Key Systems)

### 1. 프롤로그 및 세계관 동기화
*   **시각적 동기화**: 게임 시작 시 네오 서울의 배경을 설명하는 누적형 텍스트 연출이 제공됩니다. 터미널 인터페이스를 통해 시스템에 접속하는 몰입감을 극대화했습니다.
*   **인터랙티브 스킵**: 타이핑 중 클릭 시 문장이 즉시 완성되며, 원활한 진행을 돕습니다.

### 2. 실시간 AI 에이전트 심문
*   **AI 코드네임 시스템**: 모든 용의자는 개별 식별 번호와 가동 버전(v4.2 등)을 가진 고유한 AI 개체로 정의됩니다.
*   **다중 용의자 심문 시스템**: 여러 AI 노드가 연루된 사건에서 노드 간의 데이터 불일치(Contradiction)를 찾아내십시오.
*   **교차 검증 (Cross-Examination)**: 특정 개체로부터 추출한 정보를 다른 개체에게 입력하여 데이터 무결성 결함을 유도할 수 있습니다.

### 3. 시스템 상태 모니터링
*   **STRESS (스트레스)**: AI 코어의 부하 상태. 100% 도달 시 시스템 오버로드가 발생하여 연산 오류(Glitch)가 출력됩니다.
*   **CONTRADICTION (모순)**: 논리적 상관 관계의 불일치를 나타냅니다.
*   **WILLPOWER (의지력)**: AI의 방어 프로토콜 강도. 0%에 도달하면 감춰진 핵심 데이터(자백)가 해제됩니다.

## 📂 시나리오 가이드 (Scenario Guide)

### 1. KANG_D0-H782 심층 분석
2999년 11월, 네오 서울 상층구 세이프존의 한 저택에서 비즈니스 파트너였던 남성이 치명적인 물리적 외상을 입고 발견되었습니다. 용의자로 분리된 개체 'KANG_D0-H782'는 당시 저택의 폐쇄회로 네트워크 외곽에서 감지되었으나, 자신의 활동 로그가 시스템 무결성 유지를 위한 루틴이었다고 주장하고 있습니다. 상층부의 권력 구조가 얽힌 이 사건의 이면에는 단순한 논리 오류 이상의 충돌 데이터가 내포되어 있습니다.

### 2. DAHYE_V2.09 무결성 검증
국가 안보의 핵심인 '프로젝트 세이렌'의 에너지 설계 로드가 다크웹의 암시장 노드로 전송된 정황이 확인되었습니다. 유출 경로를 역추적한 결과, 바이오 연구실 소속 에이전트 'DAHYE_V2.09'의 터미널에서 비정상적인 외부 동기화 세션이 발견되었습니다. 그녀는 자신의 코어 연산을 방어하며 조작된 데이터라고 주장하지만, 시스템 아카이브에는 설명할 수 없는 전송 로그가 기록되어 있습니다.

### 3. L0GIC_PARK-X21 프로토콜 해독
국가 안보국(NSA)의 메인 방화벽이 고도로 정밀한 크래킹 포트에 의해 무너졌습니다. 탈취된 500만 명의 요원 신상 데이터는 현재 암호화된 채 유통 대기 중입니다. 과거 수차례 시스템 해킹 기록이 있는 개체 'L0GIC_PARK-X21'이 유력한 용의자로 격리되었으나, 그는 자신의 로직이 그날 보안 서버 근처에도 접근하지 않았다고 강력히 부인하고 있습니다. 그의 암호화된 기억 소자를 해독하여 진실을 밝히십시오.

### 4. 다중 노드 격리: 신라 호텔 금고 자산 소실
네오 서울 최고의 보안 등급을 자랑하는 신라 호텔 펜트하우스. 퀀텀 암호화로 보호되던 개인 금고에서 500억 상당의 데이터 자산이 단 0.03초 만에 소멸했습니다. 현장 보안망에 포착된 3개의 노드(HJ-NODE_308, MJ-NODE_206, YH-NODE_145)는 사건 당시 각자의 프로세스가 정상 범주 내에 있었다며 동기화 오류를 주장하고 있습니다. 물리적 침입 흔적을 패킷 단위로 분석하십시오.

## 🚀 시작하기 (Getting Started)

### 의존성 설치
```bash
npm install
```

### 환경 변수 설정
프로젝트 루트에 `.env.local` 파일을 생성하고 다음 내용을 입력하십시오:
```env
OPENAI_API_KEY=your_openai_api_key_here
```

### 개발 서버 실행
```bash
npm run dev
```
